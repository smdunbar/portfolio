01. Overview

Have you ever visited a website and struggled to navigate it or learn its features?

Often, this is the result of bad design choices that do not prioritize information hierarchy or the function of spaces on the screen.

Another common issue in website design is its responsiveness. How does the webpage react when viewed on screens of varying sizes?

We can examine this issue by exploring When2Meet, a time organization platform that makes finding group meeting times easier.

At First Glance

I chose this website because I often use it for clubs and activities. I have never enjoyed the interface nor found it engaging or intuitive. By inspecting the page I also found that it does not respond well to page size changes as I hypothesized.

Let's redesign this website to make it more user-friendly and visually appealing! I will focus on improving the information hierarchy, accessibility, and responsiveness of the site. I will also ensure that the design is consistent across different screen sizes in order to offer a better user experience.

02. Observations

To understand the user experience of the website, I took notes on its memorability, efficiency, learnability, conceptual model, and accessibility

Efficiency
- The layout is overly spread out, requiring excessive scrolling to access content on the left and right sides of the page.
- Despite this, the interface is simple and easy to navigate.
Memorability
- The minimalistic design lacks contrast, color differentiation, and size variations, making it visually unmemorable.
- There is no clear visual hierarchy, making it harder to quickly identify important elements.
- Buttons and icons do not have distinct styling to indicate they are clickable, which may cause confusion.
Learnability
- The interface provides clear instructions, helping users understand how to interact with key elements such as the calendar.
- However, a brief description of the page’s purpose could improve clarity and help users understand their tasks more easily.
Conceptual Model
- The calendar feature effectively visualizes selected dates, making it intuitive for scheduling.
- However, beyond the calendar, the site lacks a clear conceptual model to guide users in understanding how different elements function together.
Accessibility Concerns
- The absence of headings and defined page regions may make it difficult for screen readers to navigate the site efficiently.
- Many elements are missing labels, which makes it harder for assistive technologies to interpret content.
- The calendar is not keyboard-accessible; users can only select dates via mouse clicks, creating usability barriers for those who rely on keyboard navigation.
- These limitations could significantly impact users with accessibility needs, leading to frustration and difficulty navigating the site.

05. Implemented Redesign

Through the study of the original site, I was able to redesign When2Meet to better suit user needs, intuition, and accessibility. It can be accessed by the following link.

https://smdunbar.github.io/whenTwoMeet/

The new website has the following considerations and features:

Usability Considerations Features
Efficiency
Dropdowns rather than text boxes allows for quick entry
Defaults in the dropdowns allow for quick entry
Not overly spread out and easily scrollable
Learnability
Clear instructions are at the top of the page
Clear headers for date and time sections
Input is dropdowns rather than text boxes, creating a structured input
Memorability
Visual hierarchy through color and size create memorable sections and easily identifiable buttons
Buttons and inputs have drop shadows to indicate interaction
Narrative Navigation
Accessibility
Color contrast is improved
State changes are added to buttons and inputs on hover
Large enough text size
Font, color, and size changes offer a clear information hierarchy
Conceptual Model
Date selection is in the form of a calendar to ease user interaction
06. Conclusion

Through this project, I learned the importance of information hierarchy and accessibility in web design. By focusing on the user's interactions with the site, I was able to create an efficient, memorable, learnable, and accessible redesign of the original website.

I also learned how to use flexbox and media queries to create a responsive design. I am proud of the final product and hope it will be more user-friendly and visually appealing than the original site.
